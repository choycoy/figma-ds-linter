# Design System Linter (Figma Plugin)

Figma 파일을 스캔해서 **디자인 시스템에서 벗어난 항목**을 찾아주는 플러그인입니다.
판단 기준(source of truth)은 파일에 등록된 **Figma 변수(Variables)와 스타일(Styles)** 이고, 위반에 대한 해결책은 OpenAI가 추천하며 클릭 한 번으로 적용까지 됩니다.

## 감지 항목

| 종류 | 위반으로 판단하는 경우 |
| --- | --- |
| **색상 (Color)** | Fill·Stroke의 solid 색상이 변수(`boundVariables`)나 paint 스타일(`fillStyleId`/`strokeStyleId`)에 연결되지 않은 raw 값일 때 |
| **타이포그래피** | 텍스트가 text style에 연결되지 않았거나, 부분 적용(mixed)일 때 |
| **맞춤법** | 텍스트 노드의 내용에 맞춤법·띄어쓰기·문법 오류가 있을 때 (OpenAI로 검사) |

## AI 추천 해결책 (OpenAI)

검사 실행 시 **맞춤법 검사는 자동으로** 함께 돌아갑니다. 색상·타이포 위반에 대한 해결책 추천은 자동으로 실행되지 않으며, 결과 화면에서 **✨ AI 추천 받기** 버튼을 눌러야 실행됩니다.

- **정확히 일치하는 토큰이 있으면 API 호출 없이 즉시** 추천합니다 — hex가 완전히 같은 색상 변수/paint 스타일, family·style·size가 완전히 같은 text 스타일이 있는 경우.
- 정확히 일치하는 게 없는 나머지만 OpenAI에 보냅니다. 이때 **파일에 실제로 정의된 토큰**(색상 변수·paint 스타일·text 스타일) 목록을 함께 보내므로, 존재하지 않는 토큰을 추천하지 않습니다.
- 추천된 토큰이 파일에 있으면 **`<토큰> 연결`** 버튼으로 노드에 바로 바인딩합니다.
- 일치하는 토큰이 없으면 LLM이 기존 네이밍 컨벤션을 따르는 새 이름(예: `color/gray-900`)을 제안하고, **`새로 만들고 연결`** 버튼이 로컬 변수/스타일을 생성한 뒤 노드에 연결합니다. 같은 색을 쓰는 페이지 내 다른 노드까지 한 번에 함께 연결됩니다.
- 맞춤법 검사는 검사 실행과 동시에 자동으로 돌아가며, 오류가 발견된 텍스트에는 교정 제안과 **`"..."로 수정`** 버튼이 바로 표시됩니다. (맞춤법 교정은 노드마다 텍스트가 달라 일괄 적용 대상에서는 제외됩니다.)
- 모든 적용은 클릭 시에만 일어나며 Figma의 **Undo(⌘Z)** 로 되돌릴 수 있습니다.

### API 키 설정

이 플러그인은 **각 사용자가 자기 OpenAI API 키를 직접 입력**하는 구조입니다. 소스 코드나 배포 번들에 키를 하드코딩하지 않습니다 — Figma 플러그인은 `dist/code.js` + `dist/ui.html` 정적 파일로 배포되는데, 여기에 키를 박아두면 파일을 받은 누구나 그대로 추출해갈 수 있기 때문입니다.

1. 플러그인 우측 상단 **⚙** → OpenAI API 키 입력 → **저장**.
2. 키는 이 기기의 `figma.clientStorage` 에만 저장되고, OpenAI(`api.openai.com`) 호출에만 쓰입니다. 다른 곳으로 전송되거나 레포/번들에 포함되지 않습니다.
3. `manifest.json` 의 `networkAccess.allowedDomains` 에 `https://api.openai.com` 이 등록되어 있어야 합니다(기본 포함).
4. 모델은 `src/ai.ts` 의 `MODEL`(기본 `gpt-4o-mini`)에서 변경할 수 있습니다.

### ⚙ 설정 — 프레임 지정 (색상/타이포 각각 별도)

플러그인 우측 상단 **⚙** 에서 지정하는 두 종류의 프레임은 서로 목적이 다릅니다.

| 설정 | 용도 |
| --- | --- |
| **기준 프레임** (색상/타이포) | AI 추천용 **토큰 카탈로그를 수집할 범위**를 좁힙니다. Foundations 페이지의 컬러 팔레트/타입 스케일 섹션을 지정해두면, 그 안에서 실제 쓰이는 변수/스타일만 카탈로그에 포함됩니다. 지정하지 않으면 현재 페이지 전체를 훑습니다. |
| **카드 템플릿** (스와치/타이포 샘플) | **새 토큰을 만들 때** 디자인 시스템 문서에 추가할 카드 1개를 지정합니다. 일치하는 토큰이 없어 새로 만드는 경우, 이 카드를 복제하고 라벨(이름/HEX/RGB/폰트 스펙)만 새 값으로 채워 같은 자리에 추가합니다. |

카드 템플릿을 지정해두면 **⚙ → 카드 없는 변수/스타일 추가** 버튼으로, 아직 문서 카드가 없는 기존 변수·스타일에 대해 카드를 일괄 생성할 수도 있습니다.

## 개발 환경

```bash
npm install
npm run dev     # esbuild watch 모드
npm run build   # dist/ 에 code.js + ui.html 생성
npm run typecheck
```

## Figma에 불러오기

1. `npm run build` 로 `dist/` 생성
2. Figma 데스크톱 앱 → **Plugins → Development → Import plugin from manifest…**
3. 이 레포의 `manifest.json` 선택
4. 캔버스에서 **Plugins → Development → Design System Linter** 실행

## 사용법

- **검사 범위**: `현재 페이지` 전체 또는 `선택 영역`만
- **검사 항목**: 색상 / 타이포그래피 / 맞춤법 중 선택
- **검사 실행** 후 위반 목록이 종류별로 표시됩니다. 항목을 클릭하면 캔버스에서 해당 노드로 이동·선택됩니다.

## 구조

```
manifest.json      플러그인 매니페스트
build.mjs          esbuild 빌드 (code 번들 + UI를 단일 HTML로 인라인)
src/
  shared.ts        메인 스레드 ↔ UI 공유 타입/메시지
  code.ts          Figma 씬 그래프 순회 + 위반 감지 + 토큰 카탈로그/적용 (메인 스레드)
  ai.ts            OpenAI 호출 → 색상·타이포 토큰 추천 + 맞춤법 검사 (UI iframe)
  ui/
    ui.tsx         React UI (iframe)
    ui.css         스타일 (Figma 테마 변수 사용)
    ui.html        UI 템플릿
```
