# Design System Linter 사용 가이드

Figma 파일에서 디자인 시스템(변수·스타일)을 벗어난 색상·타이포그래피·맞춤법 오류를 찾아내고,
AI 추천을 받아 클릭 한 번으로 고치는 플러그인입니다. 이 문서는 실제 사용 흐름을 기준으로
화면별 기능을 설명합니다.

## 1. 시작하기

1. Figma 데스크톱 앱에서 **Plugins → Development → Design System Linter**를 실행합니다.
   (플러그인이 목록에 없다면 개발 환경 설정이 필요합니다 — 담당 개발자에게 문의하세요.)
2. 플러그인 창이 뜨면 우측 상단에 **⚙ 설정** 버튼이 보입니다. API 키나 프레임/카드 템플릿이
   아직 지정되지 않았다면 이 버튼이 경고 색으로 강조되어 표시됩니다.

## 2. 최초 1회 설정 (⚙)

설정 항목은 한 번만 해두면 이후에는 계속 재사용됩니다. **⚙** 버튼을 눌러 설정 화면으로 들어갑니다.

### 2-1. OpenAI API 키

- AI 추천·맞춤법 검사 기능을 쓰려면 본인의 OpenAI API 키가 필요합니다.
- **OpenAI API 키** 입력창에 `sk-`로 시작하는 키를 붙여넣고 **저장**을 누릅니다.
- 키는 이 기기에만 저장되며(`figma.clientStorage`), OpenAI 서버 호출 외에는 사용되지 않습니다.
- 저장된 키는 마스킹되어(`설정됨 (sk-ab…wxyz)`) 표시되고, 필요하면 **삭제** 링크로 지울 수 있습니다.
- API 키 없이도 색상/타이포그래피/구조 검사(스캔) 자체는 가능하지만, AI 추천과 맞춤법 검사는
  동작하지 않습니다.

### 2-2. 카드 템플릿 지정 (색상 / 타이포그래피)

디자인 시스템 문서 페이지에 컬러 스와치나 타이포 샘플을 "카드" 형태로 정리해두는 경우가 많습니다.
새 토큰을 만들 때 이 카드를 복제해서 자동으로 채워 넣기 위한 설정입니다.

1. 캔버스에서 기존 카드(예: 색상 스와치 1개, 또는 타이포 샘플 1개)를 **1개만** 선택합니다.
2. 설정 화면에서 **선택한 카드를 템플릿으로 지정**을 누릅니다.
3. 상태 표시가 `현재: <카드 이름>`으로 바뀝니다. 잘못 지정했다면 **해제**로 취소할 수 있습니다.
4. 색상용/타이포그래피용 템플릿은 서로 독립적으로 지정합니다.

> 카드 템플릿을 지정하지 않아도 검사·연결 기능은 정상 동작합니다. 다만 "새 토큰을 만들 때
> 문서에 카드를 자동으로 추가"하는 기능만 비활성화됩니다.

### 2-3. 기준 프레임 지정 (색상 / 타이포그래피)

AI가 추천할 때 참고하는 "토큰 카탈로그"의 수집 범위를 좁히는 설정입니다. 예를 들어
Foundations 페이지에 컬러 팔레트 섹션이 따로 있다면, 그 프레임만 지정해두면 AI가 그 안에서
실제로 쓰이는 변수/스타일만 추천 후보로 사용합니다.

1. 캔버스에서 기준으로 삼을 프레임을 선택합니다.
2. **선택한 프레임으로 지정**을 누릅니다.
3. 지정하지 않으면 현재 페이지 전체를 훑습니다(`미지정 (현재 페이지 전체)`).

설정을 마쳤으면 **닫기**를 눌러 메인 화면으로 돌아갑니다.

## 3. 검사 실행하기

메인 화면에서:

1. **검사 범위**를 고릅니다 — `현재 페이지` 전체 또는 캔버스에서 미리 선택한 `선택 영역`.
2. **검사 항목**을 고릅니다 — `색상`, `타이포그래피`, `맞춤법` (기본은 모두 체크).
3. **검사 실행**을 누릅니다. 검사 중에는 `검사 중…` 스피너가 표시됩니다.
4. 검사가 끝나면 상단에 `N개 위반 · N개 노드 검사됨` 요약이 뜨고, API 키가 설정돼 있다면
   색상/타이포/맞춤법 AI 추천도 자동으로 함께 돌아갑니다.

## 4. 위반 항목 확인 & 이동

- 결과는 필터 칩(`전체`, `색상`, `타이포그래피`, `맞춤법`)으로 종류별 개수를 보며 좁혀볼 수 있습니다.
- 목록의 항목을 클릭하면 캔버스에서 해당 노드로 자동 이동·선택됩니다.
- **전체 하이라이트** 버튼을 누르면 현재 필터에 표시된 모든 위반 노드를 한 번에 캔버스에서
  선택할 수 있습니다.

## 5. AI 추천으로 한 번에 고치기

검사 실행 시 자동으로 돌아가지만, 다시 실행하고 싶다면 **✨ AI 추천 받기** 버튼을 누릅니다
(색상/타이포 위반이 있을 때만 노출). 진행 중에는 `AI 추천 생성 중… (완료/전체)`로 진행률이 표시됩니다.

추천은 위반 항목 아래 💡 로 표시되며, 상황에 따라 버튼이 달라집니다.

| 상황 | 표시되는 추천/버튼 |
| --- | --- |
| 정확히 일치하는 색상 변수/스타일이 있음 | `<토큰명> 연결` — API 호출 없이 즉시 바인딩 |
| 정확히 일치하는 텍스트 스타일이 있음 | `<스타일명> 적용` |
| 일치하는 토큰이 없음 (AI가 새 이름 제안) | 이름 입력창(예: `color/blue-900`, `text/heading-01`) + **변수 만들고 + 카드 추가** / **텍스트 스타일 만들고 적용** |
| 맞춤법 오류 | `"<교정 문구>"로 수정` |

- 추천된 이름이 마음에 안 들면 입력창에서 직접 원하는 이름으로 바꿔 만들 수 있습니다.
  (AI 추천 없이도 처음부터 직접 이름을 입력해 새 토큰을 만들 수 있습니다.)
- 같은 색을 쓰는 다른 노드가 더 있으면 버튼에 `(+N개)`가 붙어, 한 번의 클릭으로 함께
  연결/적용됩니다.
- 적용 중에는 스피너가, 완료되면 **✓ 적용됨**이 표시됩니다.
- 마음에 안 들면 언제든 **Cmd+Z(Undo)** 로 되돌릴 수 있습니다.

## 6. 여러 항목 한꺼번에 적용하기

체크박스로 여러 위반(맞춤법 제외)을 선택하면 화면 하단에 선택 바가 나타납니다.

1. 고칠 항목들을 체크합니다. `N개 체크됨` 이 표시됩니다.
2. 이름 입력창에 적용할 토큰 이름을 입력합니다.
3. **체크한 N개에 이 이름으로 적용**을 누르면 선택한 모든 항목에 동일한 토큰이 한 번에
   연결/생성됩니다.
   - 단, 체크한 항목에 색상과 타이포그래피가 섞여 있으면 일괄 적용이 비활성화되고
     안내 문구가 표시됩니다. 같은 종류끼리만 묶어서 적용하세요.
4. **선택 해제**로 체크를 초기화할 수 있습니다.

## 7. 위반 무시(숨기기)

디자인 의도상 예외로 둬야 하는 항목은 무시 목록에 넣어 다음 검사부터 제외할 수 있습니다.

- 개별 항목의 **×** 버튼을 누르면 해당 위반만 무시됩니다.
- 여러 개를 체크한 뒤 **선택 항목 제거**를 누르면 한 번에 무시 목록에 추가됩니다.
- 무시된 항목이 있으면 상단에 `숨긴 항목 N개`가 표시되고, **모두 표시**를 누르면 무시 목록이
  초기화되어 다시 보이게 됩니다.
- 무시 목록은 파일에 저장되어 유지되므로, 다음에 플러그인을 다시 열어도 그대로 적용됩니다.

## 8. 문서 카드 일괄 생성

카드 템플릿(2-2)을 지정해두면, 이미 파일에 존재하지만 아직 문서에 카드가 없는 변수·스타일에
대해 카드를 한 번에 만들 수 있습니다.

- **⚙ → 카드 없는 변수 추가**: 카드가 없는 색상 변수들을 템플릿 카드 형태로 일괄 생성.
- **⚙ → 카드 없는 스타일 추가**: 카드가 없는 텍스트 스타일들을 템플릿 카드 형태로 일괄 생성.
- 카드 템플릿이 지정되지 않은 경우 이 버튼들은 비활성화 상태입니다.

## 9. 자주 묻는 질문 / 문제 해결

**Q. ⚙ 버튼이 계속 경고 색으로 표시돼요.**
A. API 키, 카드 템플릿, 기준 프레임 중 하나 이상이 아직 지정되지 않았다는 뜻입니다. 필수는
아니지만, AI 추천/카드 자동 생성 기능을 쓰려면 설정을 완료하세요.

**Q. AI 추천 받기 버튼이 안 보여요.**
A. 색상/타이포그래피 위반이 하나도 없을 때는 표시되지 않습니다. 맞춤법 위반만 있는 경우에도
숨겨집니다.

**Q. 새로 만들고 연결했는데 문서에 카드가 안 생겼어요.**
A. 카드 템플릿이 지정되지 않았을 가능성이 높습니다. ⚙ 에서 카드 템플릿을 지정한 뒤 다시
시도하세요. 템플릿이 없어도 변수/스타일 자체는 정상적으로 생성·연결됩니다.

**Q. 검사 범위를 "선택 영역"으로 했는데 결과가 0건이에요.**
A. 캔버스에서 프레임/노드를 먼저 선택한 뒤 검사를 실행해야 합니다. 아무것도 선택하지 않은
채로는 검사할 대상이 없습니다.

**Q. 잘못 적용했어요.**
A. Figma의 **Cmd+Z(Undo)** 로 바로 되돌릴 수 있습니다. 무시 처리한 항목을 되돌리려면
**모두 표시** 버튼을 사용하세요.

**Q. API 키는 안전한가요?**
A. 키는 이 기기의 로컬 저장소에만 남고, OpenAI(`api.openai.com`) 호출 외에는 어디에도
전송되지 않습니다. 다른 팀원과 파일을 공유해도 키는 함께 공유되지 않습니다(각자 본인 기기에
직접 입력해야 합니다).
